## Описание
Простое CLI-приложение для практики модуля 3. Помощник отвечает только по теме поиска багов, умеет классифицировать обращения (FAQ или проблема) , кешировать (одинаковые вопросы), делать retry (tenacity) и fallback, логировать диалог, распознает изображения и поддерживает базовые команды. Так как прямое назначение помощника - классифицировать баги, то при FAQ запросах мы не обращаемся к модели за поиском багов и не тратим токены, а отдаем залушку "Запрос не относится к проблемам. Найдите ответ в базе знаний"

## Структура
```text
m2_b5/

  cli.py                  # точка входа
  
  models.py               # все dataclass-ы и type alias-ы
  
  core/
  
    assistant.py          # сценарий диалога и команды CLI
    
    classification.py     # категории
    
  infrastructure/

    cache.py              # пока не удалось установить Redis
    
    llm.py                # retry (tenacity) + fallback для LLM

    tools.py              # вызов функции search_bug_database
    
  prompts/
  
    loader.py             # загрузка prompt-файлов
    
    system_prompt.txt
    
    classifier_system_prompt.txt
    
    classifier_category_few_shots.json
    
    bugs_database.json

    tool_search_bugs_description.txt

```

## Запуск

Из папки: 
```bash
python -m m2_b5
```
## Прочее
BAG_SERVICE_NAME — имя сервиса, по умолчанию EVA/

Без ключей приложение тоже работает: классификация работает эвристикой, при недоступности обоих провайдеров — заглушка "Извините, что-то пошло не по плану. Пожалуйста, повторите попытку или напишите чуть позже."
